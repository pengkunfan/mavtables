20 # Installing
21 
22 ## Requirements
23 
24 In order to compile you will need the following packages:
25 
26 * GCC 7+ or Clang 5+ (needed for C++17 support)
27 * [CMake v3.3+](https://cmake.org/)
28 * [Boost v1.54+](https://www.boost.org/)
29 
30 on either
31 
32 * Linux
33 * Mac OS X
34 * BSD
35 
36 mavtables is tested on both Linux and Mac OS X and should work on any unix
37 compatible system.
38 
39 
40 ## CMake Installation
41 
42 To install via CMake simple run:
43 ```
44 $ make
45 $ make install
46 ```
47 
48 The installation prefix is `/usr/local` by default but can be changed with
49 ```
50 $ make PREFIX=/desired/install/path
51 # make DESDIR=/desired/install/path install
52 ```
53 `PREFIX` changes where mavtables expects to be installed while `DESTDIR` changes
54 where it will actually be installed.
55 
56 To change the MAVLink dialect use the `DIALECT` environment variable. For
57 example, to use the `common` dialect:
58 ```
59 $ make DIALECT=common
60 # make install
61 ```
62 
63 To change the MAVLink implementation completely use the `MDIR` environment
64 variable to set the path to the MAVLink library instead of downloading the
65 upstream implementation. For example, if a custom implementation is at
66 `/tmp/mavlink/v2` then:
67 ```
68 $ make MDIR=/tmp/mavlink/v2
69 # make install
70 ```
71 Both the `MDIR` and `DIALECT` variables can be used together.

74 ## Manual Installation
75 
76 To manually install mavtables first make a release build:
77 ```
78 $ make
79 ```
80 Remembering to use the `PREFIX` option if the final installation destination is
81 not `/usr/local`.
82 
83 Copy binary, configuration, and systemd unit files to their proper locations
84 ```
85 # cp build/mavtables /usr/local/bin/
86 # cp examples/mavtables.conf /usr/local/etc/
87 # cp build/mavtables.service /usr/local/lib/systemd/system/
88 ```

91 # Usage
92 
93 ## Running
94 
95 To run mavtables and begin routing packets
96 ```
97 $ mavtables
98 ```
99 This will use the first configuration file it finds in the configuration file
100 priority order:
101 
102 1. The target of the `MAVTABLES_CONFIG_PATH` environment variable.
103 2. `.mavtablesrc` in the current directory.
104 3. `.mavtablesrc` at `$HOME/.mavtablesrc`.
105 4. The main configuration file at `/etc/mavtables.conf`.
106 
107 To specify the configuration file use
108 ```
109 $ mavtables --config path/to/config/file
110 ```
111 
112 If the configuration file contains an error, mavtables will exit immediately and
113 print an error message to stderr.
114 
115 The installer will install a system wide configuration file at
116 `PREFIX/etc/mavtables.conf` and a systemd unit file at
117 `PREFIX/lib/systemd/system/mavtables.service`. Therefore, on Linux (with
118 systemd) mavtables can be run as a daemon with
119 ```
120 # systemctl start mavtables
121 ```
122 or enabled for startup on each boot with
123 ```
124 # systemctl enable mavtables
125 ```
126 
127 systemd will keep mavtables running (even if it crashes) and will use the
128 `PREFIX/etc/mavtables.conf` configuration file. It will also use `loglevel` 1,
129 discussed in the [Log Level](#log-level) section below.

132 ## Log Level
133 
134 The `--loglevel` flag can be use to set the logging level. This is the
135 verbosity to print to stdout when mavtables is running. Each loglevel,
136 0 through 3, is documented below.
137 
138 ### --loglevel 0
139 
140 Do not log anything to stdcout.
141 
142 ### --loglevel 1
143 
144 Log each new component. Therefore, every time a new system/component address is
145 connected to the router it will be printed to stdcout. An example log output
146 is:
147 ```
148 new component 127.1 on 127.0.0.1
149 new component 192.168 on 127.0.0.1
150 new component 172.128 on ./ttyS0
151 new component 10.10 on 127.0.0.1
152 ```
153

217 ## Abstract Syntax Tree
218 
219 mavtables can print the abstract syntax tree of the configuration file instead
220 of routing packets. When run in this mode it will return immediately This is
221 accomplished with:
222 ```
223 $ mavtables --ast
224 ```
225 or
226 ```
227 $ mavtables --ast --config path/to/config/file
228 ```
229 which will print something similar to
230 ```
231 ===== /etc/mavtables.conf =====
232 :005: default_action
233 :005: | reject
234 :008: udp
235 :009: | port 14500
236 :010: | address 127.0.0.1
237 :022: chain default
238 :023: | accept
239 ```
240 
241 This feature can be used to debug configuration files.
242
