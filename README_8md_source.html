5 
6 A MAVLink router and firewall. It can connect 2 or more MAVLink endpoints such
7 as autopilots, ground control software, loggers, image capture systems, etc over
8 serial and UDP. MAVLink packets will be routed to specific components when they
9 have a destination address. Any packet, targeted or broadcasted, can be
10 filtered based on source system/component, destination system/component and
11 message type. The filter can also apply a priority to packets allowing more
12 important packets to take priority over lower priority packets when an endpoint
13 is choked.

26 ## Compilation and Installation
27 
28 In order to compile you will need the following packages:
29 
30 * GCC 7+ or Clang 5+ (needed for C++17 support)
31 * [CMake v3.3+](https://cmake.org/)
32 * [Boost v1.54+](https://www.boost.org/)
33 
34 __Clang is recommended when contributing to mavtables as it's warnings are
35 more comprehensive. However, GCC must be used when generating code coverage
36 reports.__
37 
38 The following packages are only needed for development work:
39 
40 * [Artistic Style](http://astyle.sourceforge.net/) (used for
41  checking/fixing the code style)
42 * [Gcovr](http://gcovr.com/) (coverage report)
43 * [LCOV](http://ltp.sourceforge.net/coverage/lcov.php) (detailed coverage html
44  report)
45 * [socat](http://www.dest-unreach.org/socat/) (for testing serial port
46  communications)
47 
48 `mavtables` can be easily installed using the standard procedure of
49 ```
50 $ make
51 # make install
52 ```
53 The installation prefix is `/usr/local` by default but can be changed with
54 ```
55 $ make
56 # make PREFIX=/desired/install/path install
57 ```
58 The makefile will download and use the default MAVLink implementation with the
59 ArduPilot dialect. This can be overridden by setting the `MDIR` environment
60 variable to the library path (containing the `DIALECT`) and/or the `DIALECT`
61 environment variable to the MAVLink dialect to use. For instance the default
62 value of `DIALECT` is `ardupilotmega`.

65 ## Running
66 
67 To run mavtables and begin routing packets
68 ```
69 $ mavtables
70 ```
71 This will use the first configuration file it finds in the configuration file
72 priority order given in the next section. To force a specific configuration
73 file the `--config` flag may be used.
74 ```
75 $ mavtables --config <path/to/config>
76 ```
77 The inbuilt help may be accessed with the `-h` or `--help` flags.
78 ```
79 $ mavtalbes --help
80 usage: mavtables:
81  -h [ --help ] print this message
82  --config arg specify configuration file
83  --ast print AST of configuration file (do not run)
84  --version print version and license information
85  --loglevel arg level of logging, between 0 and 3
86 ```

89 ## Configuration File
90 
91 Both interfaces and filter rules are defined in a configuration file. The
92 format of this configuration file is documented in
93 [`doc/configuration.md`](doc/configuration.md) and an example is located at
94 `examples/mavtables.conf` which is the same file that is installed at
95 `/etc/mavtables.conf` when using `make install`. The configuration file used is
96 the first one found in the following order:
97 
98 1. The target of the `MAVTABLES_CONFIG_PATH` environment variable.
99 2. `.mavtablesrc` in the current directory.
100 3. `.mavtablesrc` at `$HOME/.mavtablesrc`.
101 4. The main configuration file at `/etc/mavtables.conf`.
102 
103 If the `--config` flag is given then mavtables will only look for the given
104 configuration file.
